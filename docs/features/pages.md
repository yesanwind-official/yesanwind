## 1. 홈페이지 (메인)

### 1.1 기본 정보

| 항목 | 내용 |
|------|------|
| 기능 ID | PAGE-001 |
| 페이지명 | 홈페이지 |
| URL | `/` |
| 우선순위 | Must Have |
| 목적 | 오케스트라 첫인상 제공, 주요 정보 요약 노출 |

### 1.2 사용자 스토리

**As a** 웹사이트 방문자,
**I want to** 메인 페이지에서 오케스트라의 핵심 정보를 한눈에 파악하고 싶다,
**So that** 관심 있는 섹션으로 빠르게 이동할 수 있다.

### 1.3 주요 컴포넌트

| 컴포넌트 | 설명 | 기능 |
|----------|------|------|
| Hero Section | 메인 비주얼 영역 | 대표 이미지/영상, 슬로건, CTA 버튼 |
| 공연 안내 섹션 | 다가오는 공연 | 최근 예정 공연 1-3개 카드 노출 |
| 오케스트라 소개 섹션 | 간략 소개 | 소개 텍스트, 더보기 링크 |
| 갤러리 미리보기 | 최근 사진 | 최근 갤러리 이미지 4-6개 |
| 공지사항 섹션 | 최근 공지 | 최근 공지사항 3-5개 목록 |
| Footer | 푸터 | 연락처, SNS, 저작권 |

### 1.4 화면 상태

| 상태 | 조건 | 표시 내용 |
|------|------|-----------|
| 기본 상태 | 데이터 정상 로드 | 모든 섹션 정상 표시 |
| 공연 없음 | 예정 공연 없음 | "예정된 공연이 없습니다" 메시지 |
| 갤러리 없음 | 등록된 사진 없음 | 섹션 숨김 또는 기본 이미지 |
| 로딩 중 | 데이터 로딩 | 스켈레톤 UI |
| 오류 | API 오류 | 에러 메시지 및 새로고침 버튼 |

### 1.5 수락 기준

```gherkin
Scenario: 메인 페이지 접속
  Given 사용자가 웹사이트에 접속한다
  When 메인 페이지가 로드된다
  Then Hero 섹션이 표시된다
  And 다가오는 공연이 있으면 공연 정보가 표시된다
  And 최근 공지사항이 표시된다

Scenario: 공연 카드 클릭
  Given 메인 페이지에 공연 카드가 표시된다
  When 사용자가 공연 카드를 클릭한다
  Then 해당 공연 상세 페이지로 이동한다
```

```gherkin
Scenario: 존재하지 않는 공연
  Given 존재하지 않는 공연 ID로 접속한다
  When 페이지가 로드된다

```
